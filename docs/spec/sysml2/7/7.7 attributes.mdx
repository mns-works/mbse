### 7.7.1 属性概述

*属性定义（attribute definition）*可以定义数据值，
例如数字，带有单位的数值，定性的值（例如文本字符串），或者这些值组成的数据结构。
*属性使用（attribute usage）*是一个属性定义的使用。
属性使用一定是引用的，而且属性定义或使用的任何内嵌特征也必须是引用的。

属性使用的数据值需要在它的定义所指定的范围内。
属性定义中数据值的范围可以进一步使用约束来限制。
枚举定义是一种特种的属性定义，其值是一系列离散的数据值。

属性使用可以被 KerML 的数据类型以及 SysML 的属性定义所定义。
这使得属性使用的类型可以是 Kernel 的 DataType 库中的基础数据类型，
例如`String`，`Boolean`，以及包括`Integer`、`Rational`、`Real`和`Complex`在内的数字类型。
Kernel 的 DataType 库也包括描述集合的基本结构数据类型。

表示带有单位的数量的属性使用，则通过使用 SysML 的数量和单位领域库以及这个库中元素的扩展来定义。
这个库中提供了对标量、矢量以及张量的基本属性定义，
还包括其他实现了完整国际标准度量类型和单位的模型元素。
这种方式的基本原则是在属性定义中只关联单位的种类（例如，重量单位`MassUnit`，长度单位`LengthUnit`，时间单位`TimeUnit`等），
而具体的单位只和实际的数值一起给出。
这意味着表示数量值的属性使用和所使用的特定的单位是独立的，
使得可以在相同种类的不同单位之间可以自动进行转换（例如，千克和磅，米和英尺）。

属性使用的值是数据值，不管是基础值（例如整数）还是滴啊有内嵌属性的结构值，这些值本身都不随时间变化。
然而，属性使用的所有者可能是实存定义或使用（可能是特化的实存，例如项目、部件或动作）。
这种情况下，属性使用的值可能在其所属实存的寿命中会变化，
看起来就像是在不同的时间点上具有不同的值，
这反映了实存随时间而不断变化的状况。

### 7.7.2 属性定义和使用

属性定义通过关键字`attribute`来声明。
属性定义只能特化其他的属性定义（包括枚举定义）或者KerML中的数据类型。
属性使用通过关键字`attribute`来声明。 
属性使用是能被属性定义或者kerML的数据类型所定义。

属性使用总是引用类型的，无论是否显示的使用`ref`关键字。 
属性使用的默认多重性是`[1..1]`。
一般而言，任何种类的使用都可以在属性定义或属性使用的内容体中声明，
但是这些使用都应该是引用的。

```sysml 
    attribute def SensorRecord {
        ref part sensor : Sensor;
        attribute reading : Real;
    }
```

基本的属性定义是`Attributes`库中的`AttributeValue`，其只是KerML`Base`库中的
数据类型`DataValue`的一个别名。
基本的属性使用是`Attributes`库中的`attributeValues`，其只是KerML`Base`库中的特征
`dataValues`的一个别名。
