*连接定义*既是一种关系，又是一种部件定义，用来对相关联的事物之间的连接（例如项目和部件）进行分类。
连接定义的最后两个拥有的特征必须是*连接端（connection ends）*，用于指定该连接定义所关联的事物。
只具有两个连接端的连接定义称为*二元连接定义（binary connection definition）*，其对*二元连接（binary connection）*进行分类。

连接定义的特征中，不属于连接端的特征
由于连接也是部件，这些非端特征的值也可能随时间变化。
然而，连接端的值（即实际连接的事物）不随时间变化（尽管这些值本身有可能是具有随时间变化的特征值的实存）。

**绑定和次序**
绑定和次序属于特殊的连接。它们不像一般的连接使用那样属于部件使用。
由绑定和次序指定的连接并不属于实存，因此不能被创建和销毁。
它们只是断言由其所连接的特征之间的关系必须总是为真。

*绑定（binding）*是一种二元连接，需要它所关联的使用具有相同的值。
绑定可以用来将一个上下文中的引用特征绑定到另一个上下文中的复合特征上，
来断言它们是同一个事物。
例如，汽车中的方向盘可以认为是汽车内部的一个部件，同时也可以认为是驾驶系统的一部分。
方向盘可以同时是内部的一个复合部件使用，以及驾驶系统的一个引用部件使用，
这两个使用可以绑定在一起，来断言它们是同一个部件。

*特征值（feature value）*是一种简写形式，用于初始化，或者将一个使用绑定到一个表达式的结果上
有两种特征值绑定。

  - *固定*特征值在对使用（usage）进行声明时，将给定表达式的求解结果绑定到使用上。
    这样的绑定不能通过重定义来重载，因为一旦使用，绑定对使用的所有实例都必须是真。
  - *默认*特征值
